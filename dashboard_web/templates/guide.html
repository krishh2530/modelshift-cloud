ADD TO YOUR NOTEBOOK / SCRIPT

Paste this after your existing training code. Replace X_train, X_live, and model with your own variables.

import modelshift.monitor as ms # ── 1. Connect to your dashboard ────────────────────────────────── ms.login( email=creds["email"], password=creds["password"], dashboard_url="https://modelshift-api.onrender.com" ) # ── 2. Create monitor with your TRAINING data as baseline ───────── # X_train must be a pandas DataFrame — features only, no target monitor = ms.ModelMonitor(X_train) # ── 3. Set baseline predictions (what model predicted on train set) ─ # For classifiers : baseline_preds = model.predict_proba(X_train)[:, 1] # For regressors : baseline_preds = model.predict(X_train) monitor.set_baseline_predictions(baseline_preds) # ── 4. Feed in your LIVE / TEST data ────────────────────────────── # Same columns as X_train, same dtype — no target column monitor.update(X_live) # ── 5. Feed in live predictions ─────────────────────────────────── # live_preds = model.predict_proba(X_live)[:, 1] monitor.update_predictions(live_preds) # ── 6. Compute drift ────────────────────────────────────────────── monitor.compute_feature_drift() monitor.compute_prediction_drift() # ── 7. Beam to your dashboard ───────────────────────────────────── monitor.push() print("✅ Done! Check your dashboard.")
!

IMPORTANT CONSTRAINTS

DATA FORMAT

X_train and X_live must be pandas DataFrames with identical column names. No target/label column — features only.

PREDICTIONS

Must be probability values between 0 and 1. Use predict_proba(X)[:, 1] for classifiers. Do NOT use raw predict() for classification.

MINIMUM SIZE

Both datasets need at least 2 rows. Recommended: 100+ rows for statistically meaningful KS scores.

CALL FREQUENCY

Call monitor.push() as often as you want. Every call creates a new timestamped run in your history. Dashboard shows latest + previous automatically.

WHAT YOU SEE ON THE DASHBOARD

favorite Health Score

0–100 score. 100 = model behaves identically to training. 0 = complete distribution collapse.

sensors Feature Drift Table

Per-feature KS statistic and severity. Tells you exactly which sensor or variable broke your model.

show_chart Evaluation Metrics

Accuracy, F1, ROC-AUC, RMSE — Clean vs Drifted side by side. Degraded values highlighted in red.

notifications_active Email Alerts

If drift is CRITICAL, an email is fired instantly to your registered address with the fault feature and KS score.

table_view Dataset Inspector

15-row sample of your live vs baseline data. Drifted cells highlighted red. Private to your account only.

history Run History

Every push creates a permanent record in the database. Compare runs over time. Download reports.

COMPLETE WORKING EXAMPLE

Copy-paste ready. Works with any sklearn classifier.

import json, os import pandas as pd from sklearn.ensemble import RandomForestClassifier from sklearn.model_selection import train_test_split import modelshift.monitor as ms # ── Your existing training code ──────────────────────────────────── df = pd.read_csv("your_dataset.csv") X = df.drop(columns=["target"]) y = df["target"] X_train, X_live, y_train, y_live = train_test_split(X, y, test_size=0.3, random_state=42) model = RandomForestClassifier() model.fit(X_train, y_train) # ── ModelShift integration (add below your training code) ────────── creds = json.load(open("modelshift_auth.json")) ms.login(email=creds["email"], password=creds["password"], dashboard_url="https://modelshift-api.onrender.com") monitor = ms.ModelMonitor(X_train) monitor.set_baseline_predictions(model.predict_proba(X_train)[:, 1]) monitor.update(X_live) monitor.update_predictions(model.predict_proba(X_live)[:, 1]) monitor.compute_feature_drift() monitor.compute_prediction_drift() monitor.push() print("✅ Dashboard updated!")
